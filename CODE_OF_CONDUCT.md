# Keystone Code of Conduct

Keystone is an open-source project built around direct, evidence-based collaboration. The goal is not performative politeness. The goal is productive, respectful work.

## Expected Behavior

- be direct without being dismissive
- critique ideas, code, and reasoning instead of people
- explain claims with evidence when possible
- welcome contributors with different levels of experience
- respect boundaries around security, privacy, and confidential information

## Unacceptable Behavior

- harassment, intimidation, or personal attacks
- discriminatory language or behavior
- repeated bad-faith argument after maintainers or contributors have clarified the issue
- doxxing, credential exposure, or sharing private information without consent
- intentionally disruptive behavior in issues, PRs, or discussions

## Maintainer Responsibilities

Maintainers may edit, hide, lock, or remove comments and contributions that violate this code of conduct. They may also pause a discussion that is no longer productive.

## Reporting

If you experience or witness behavior that violates this code of conduct, contact the maintainers through the security/reporting path in [SECURITY.md](./SECURITY.md) or by opening a private channel if one is available.
